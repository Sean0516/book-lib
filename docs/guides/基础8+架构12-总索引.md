# 基础8 + 架构12 - 总索引

这份总索引把你现在已经完成的 `基础 8 模块 + 架构 12 模块` 串成一个统一入口。  
如果你后面要系统学习、按模块复习、查弱项或者做模拟面试，都可以先从这里进。

## 一、总览

- 基础模块：`8` 个
- 架构模块：`12` 个
- 当前总模块数：`20` 个
- 适用方向：`高级后端工程师 / 架构师 / 技术负责人`

18. 架构模块12 - 架构治理与技术管理
19. 基础模块4 - 计算机基础
20. 基础模块8 - 算法与数据结构基础

这样安排的原因是：
- 先把 Java 后端的语言、运行时、并发、数据库、缓存这条主线打稳
- 再进入系统设计、分布式、治理、稳定性这些架构主战场
- 计算机基础适合在中后段和架构题互相映射
- 算法保留在最后做专项冲刺，效率通常更高

## 六、每个模块怎么读

每个模块都统一包含这些部分：

- `核心结论`：先快速记结论
- `标准答案`：自然口述版，适合真实面试表达
- `追问点答案`：帮助你往下深挖
- `原理补充`：补底层机制和判断逻辑
- `架构扩展`：把知识点拉到真实系统场景
- `工程指标`：训练你带指标作答
- `证据链`：训练你排障、论证和定位思路
- `低风险优化路径`：训练你说“先做什么，再做什么”
- `易错点`：方便你查漏补缺

## 七、推荐复习节奏

如果你每天能拿出 `2 小时`，比较稳的节奏是：

1. `40 分钟` 看一个模块的 `核心结论 + 标准答案`
2. `30 分钟` 看 `追问点答案 + 原理补充`
3. `20 分钟` 看 `架构扩展 + 工程指标`
4. `20 分钟` 自己口述 `3 到 5 题`
5. `10 分钟` 记录今天最薄弱的 `1 到 2 个点`

如果你每天能拿出 `3 小时`，可以额外加：

1. `30 分钟` 做专项追问或场景题改写
2. `30 分钟` 重答昨天错得最重的题
