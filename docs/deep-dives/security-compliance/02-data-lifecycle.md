# 数据保护与生命周期

## 90 秒速答

数据保护先建立数据清单和所有者，按敏感度、用途、地域、保留期和主体权利分类；能不采集就不采集，
能令牌化就避免传播明文。传输和存储加密只是基础，密钥必须与数据分离，使用受控密钥服务、短期访问、
轮换、版本和紧急吊销。访问按用途和最小权限控制，日志、缓存、搜索、向量库、备份和分析副本都纳入
生命周期。删除请求通过可追踪任务传播到所有在线副本，备份采用到期不可恢复或加密销毁，并保存不含
原始敏感数据的完成证据。通过抽样恢复、权限审计和删除演练验证，而不是只看流程状态成功。

## 生命周期

```mermaid
flowchart LR
  A["采集/告知"] --> B["分类/用途绑定"]
  B --> C["存储/访问/加密"]
  C --> D["共享/处理/审计"]
  D --> E["归档/保留"]
  E --> F["删除/证明"]
```

用途变化应重新评估授权和告知，不能因为数据已经存在就无限复用。

## 加密与密钥边界

- 信封加密：数据密钥加密数据，主密钥保护数据密钥，便于轮换和隔离。
- 密钥不进入代码、镜像、普通配置和日志；工作负载通过短期身份获取使用权。
- 轮换要支持新写新版本、旧读旧版本和渐进重加密，不能一次停机全量改写。
- 应急吊销前评估可用性影响；密钥丢失可能等同数据永久丢失，必须备份和双人控制。

## 脱敏、哈希、令牌化

展示脱敏用于界面最小暴露；密码使用专门的慢哈希与独立盐；确定性哈希可能被字典推断；令牌化把
映射集中在受控保险库。选择取决于是否需要还原、关联查询和泄露后的影响，不可混用概念。

## 删除传播清单

主库、只读副本、缓存、搜索索引、对象存储、消息保留、日志、数仓、向量库、模型输入记录、第三方
处理方和备份都要有策略。删除任务必须幂等、可重试、可对账，并处理新副本在删除过程中的竞态。

## 面试官追问

### L1：备份不能逐条删除怎么办？

备份隔离、限定保留期，恢复时重新执行删除清单；高敏数据可按主体或批次使用可销毁密钥。需在政策中
说明恢复和最终不可恢复时间。

### L2：日志是否可以记录明文方便排查？

不能把便利当默认理由。记录必要标识和脱敏字段，敏感调试使用受控临时开关、审批、短保留和审计。

### L3：如何证明删除完成？

保存任务 ID、对象范围、各处理方状态、版本和完成时间，抽样查询与恢复演练验证；证明本身不应复制
被删除的敏感内容。
