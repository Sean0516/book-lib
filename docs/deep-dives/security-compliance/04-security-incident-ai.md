# 安全事件响应与 AI 边界

## 90 秒速答

安全事件先确认资产、身份、时间窗口和影响面，保护日志、快照与审计证据；并行做可逆止损，例如吊销
凭证、隔离工作负载、关闭高风险工具和限制数据出口。根除阶段修复初始入口与横向移动路径，恢复时
轮换受影响密钥、重建可信制品并阶梯放流，持续监控复发指标。AI 系统中把检索文档、网页和工具输出
视为不可信数据，系统策略与授权在模型外强制执行；高风险副作用需要确定性校验、审批和幂等。复盘
覆盖为何未预防、未早发现和未限制爆炸半径，并通过同路径演练验证整改。

## 事件阶段

```mermaid
flowchart LR
  A["检测与分级"] --> B["保护证据"]
  B --> C["隔离与吊销"]
  C --> D["根除入口"]
  D --> E["可信恢复"]
  E --> F["通知、复盘、演练"]
```

不要在保全证据前随意清日志或重装全部机器；也不要为了完美定因延迟凭证吊销和数据出口隔离。

## Prompt Injection 的边界

Prompt 提醒“忽略恶意指令”只是软控制。真正边界包括：内容与指令分离、检索权限过滤、工具白名单、
执行端重新授权、输出目的地限制、高风险审批、预算和审计。模型可以提出动作，不能自行扩大权限。

## 安全响应指标

| 指标 | 说明 |
| --- | --- |
| 检测时间 | 从首次恶意行为到告警 |
| 吊销传播时间 | 凭证失效到所有执行点拒绝 |
| 影响资产数 | 爆炸半径 |
| 未授权成功动作 | 控制是否真正失效 |
| 数据出口量 | 外泄规模与通知判断 |
| 同路径复测通过率 | 整改是否有效 |

## 面试官追问

### L1：发现密钥泄露先删代码还是先换密钥？

先吊销/轮换并限制影响面，同时保全证据；从 Git 历史删除不能使已泄露密钥失效。

### L2：内容过滤能否阻止 Agent 越权？

不能单独保证。工具执行端必须基于可信身份、对象和动作重新授权，并限制副作用和出口。

### L3：恢复服务后为什么还要继续监控？

攻击者可能保留会话、派生凭证、持久化入口或已投放恶意数据。需查找横向移动并观察复发信号。

| 维度 | 5 分要求 |
| --- | --- |
| 正确性 | 检测、隔离、根除、恢复、复盘顺序合理 |
| 深度 | 覆盖证据、派生凭证、AI 工具和数据出口 |
| 取舍 | 止损、可用性、取证与通知平衡 |
| 表达 | 影响 → 行动 → 证据 → 恢复 → 验证 |
| 可运维性 | 吊销、审计、演练、指标和 owner 完整 |
