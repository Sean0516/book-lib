# 容量规划与性能工程

这一模块训练的不是“会看 CPU”，而是把业务预测、排队规律、资源预算、压力模型和回归门禁
连成一条可验证的工程链。

## 学习地图

| 专题 | 核心问题 | 关键产出 |
| --- | --- | --- |
| [容量模型与安全水位](./01-capacity-model) | 要多少实例、线程、连接和故障冗余 | 假设表、资源预算、扩容阈值 |
| [真实性压测与瓶颈证据](./02-realistic-load-testing) | 压测如何接近生产并找到拐点 | 流量模型、证据链、容量曲线 |
| [尾延迟与性能回归](./03-tail-latency-regression) | 平均值正常时为什么用户仍然慢 | 延迟预算、分位数、回归门禁 |

## 一条完整的性能决策链

```mermaid
flowchart LR
  A["业务量与峰值分布"] --> B["服务需求与在途量"]
  B --> C["单实例安全容量"]
  C --> D["故障与扩容冗余"]
  D --> E["真实性压测"]
  E --> F["瓶颈证据"]
  F --> G["回归门禁与生产校准"]
  G --> A
```

## 推荐练法

1. 不看答案，先把每篇示例中的容量数字手算一遍。
2. 每个结论必须区分“观测事实、推导结果、待压测假设”。
3. 优化后同时报告吞吐、TP99、错误率、资源成本和回滚阈值。

## 完成标准

- 能从 DAU、集中系数和请求构成推导峰值 QPS。
- 能解释吞吐、响应时间、并发量与利用率之间的关系。
- 能识别均匀流量、热数据、缓存预热和闭环压测造成的失真。
- 能用证据定位 CPU、锁、GC、IO、连接池或下游等待，而不是凭经验调参。
