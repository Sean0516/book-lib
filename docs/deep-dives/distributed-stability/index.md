# 分布式系统与稳定性

这一模块不再孤立背 CAP、熔断或 SLO，而是训练三个架构师级问题：业务事实如何跨服务收敛，
局部异常如何不扩散，以及事故如何从发现、止血走到防复发。

## 学习地图

| 专题 | 要解决的问题 | 面试输出 |
| --- | --- | --- |
| [分布式一致性选择树](./01-distributed-consistency) | 强一致、消息、Outbox、Saga 怎么选 | 约束、不变量、补偿、对账 |
| [流量保护与级联故障](./02-traffic-protection) | 限流、超时、重试、熔断、隔离怎么组合 | 预算、传播链、恢复策略 |
| [SLO、演练与事故闭环](./03-slo-incident-loop) | 稳定性投入如何量化和验收 | SLI/SLO、错误预算、RTO/RPO、复盘 |

## 推荐练法

1. 先读每篇的 `90 秒速答`，尝试不看稿复述。
2. 再用文中的场景数据做决策，必须说明为什么不选另一种方案。
3. 最后完成三级追问，并按 25 分量表自评。

## 一条完整的架构推演链

```mermaid
flowchart LR
  A["业务目标与不变量"] --> B["一致性边界"]
  B --> C["容量与依赖预算"]
  C --> D["保护与降级"]
  D --> E["指标与告警"]
  E --> F["演练、恢复、复盘"]
  F --> A
```

> [!TIP]
> 面试时不要从中间件名字起手。先说业务损失、不变量和恢复承诺，再进入技术方案。

## 完成标准

- 能在 90 秒内给出结构化结论，而不是罗列组件。
- 能画出故障传播路径和数据状态机。
- 能给出阈值来源、回滚条件和验证指标。
- 能明确哪些风险接受，哪些必须通过不变量兜底。
