# 深度精选

原有 520 道题负责知识广度，这里负责工程深度。每篇文章先给出可直接口述的
`60–90 秒速答`，再展开机制、证据、代码、权衡、指标和三级追问。

## 当前模块

### [完整案例库](./cases/)

覆盖 JVM、数据库、Redis、消息队列、系统设计、AI、云原生、业务演进和安全合规，共 27 个
完整案例；每篇都提供证据链、止血方案、长期修复、指标、追问和评分标准。

### [JVM 与并发](./jvm-concurrency/)

从 GC、内存泄漏、线程池、锁竞争、JMM、CPU 排查到 Java 虚拟线程，最后通过一个 Full GC
教学案例把核心主题串成完整证据链。

### [数据与消息可靠性](./data-reliability/)

从数据库事实源、缓存副本到消息传播，训练执行计划、MVCC、热点回源、幂等、顺序、积压、
故障降级和不停机迁移的完整决策链。

### [分布式系统与稳定性](./distributed-stability/)

从业务不变量与一致性选择树出发，串联端到端 deadline、重试预算、熔断隔离、SLO、错误预算、
故障演练和事故复盘，训练从设计到运营的稳定性闭环。

### [容量规划与性能工程](./capacity-performance/)

从业务峰值推导实例、线程、连接与故障容量，通过开放模型、热点数据和阶梯压测寻找容量拐点，
再用尾延迟证据链与性能门禁防止版本回退。

### [AI 架构深度专题](./ai-architecture/)

贯通 RAG 分层质量、Agent 状态与工具安全、多模型路由、AI SLA、单位成功成本以及从离线评测
到线上灰度的发布闭环，并配套 3 个生产案例。

### [云原生与架构治理](./cloud-governance/)

从 Kubernetes 探针、资源与故障域出发，贯通渐进式交付、多集群、平台黄金路径、ADR、技术债
和自动化架构守护，并配套 3 个事故与转型案例。

### [业务建模与系统演进](./business-evolution/)

从业务事件和统一语言识别限界上下文，以不变量确定聚合与事务边界，再用绞杀者模式、反腐层、
影子校验和退场指标完成单体及遗留系统演进，并配套 3 个生产案例。

### [安全与合规工程](./security-compliance/)

从身份、对象级授权、租户隔离和数据生命周期出发，贯通密钥、供应链来源证明、威胁建模、安全
事件响应与 AI 工具边界，并配套 3 个安全事故案例。

## 按岗位学习

| 路线 | 当前建议 | 重点能力 |
| --- | --- | --- |
| 高级 Java 后端 | JVM 与并发全部 7 题 → Full GC 案例 | 原理、代码、排障 |
| 架构师/技术负责人 | GC 选型 → 线程池容量 → 锁竞争 → Full GC 案例 | 约束、权衡、指标 |
| AI 架构师 | 线程池容量 → 虚拟线程 → CPU 排查 | AI 网关和工具执行的 JVM 底座 |
| 数据架构方向 | 数据库证据链 → 缓存一致性 → 消息可靠性 | 事实源、传播、对账和恢复 |
| 分布式架构方向 | 一致性选择树 → 流量保护 → SLO 事故闭环 | 不变量、故障传播、恢复验收 |
| 性能与容量方向 | 容量模型 → 真实性压测 → 尾延迟回归 | 量化推导、证据链、持续门禁 |
| AI 架构方向 | RAG 质量 → Agent 安全 → 模型路由 → 评测门禁 | 任务质量、安全、SLA 与成本 |
| 云原生治理方向 | K8s 可靠性 → 渐进交付 → 平台工程 → 架构守护 | 故障域、交付安全、组织演进 |
| 业务架构方向 | 领域发现 → 聚合不变量 → 单体迁移 → 遗留治理 | 业务边界、数据所有权、系统演进 |
| 安全架构方向 | 身份授权 → 数据生命周期 → 供应链 → 事件响应 | 信任边界、纵深防御、证据与恢复 |

后续会按质量验收结果增加综合模拟面试、学习计划和掌握度反馈；不会为了数量复制同一种模板。

## 你会在文章里看到什么

- **图**：机制图、时序图和排查决策树。
- **代码**：能暴露边界和风险的最小 Java、命令或配置。
- **案例**：带输入数据、时间线、证据与回滚方案的教学场景。
- **指标**：说明定义、来源、示例基线和决策动作。
- **评分**：正确性、深度、权衡、表达和可运维性，共 25 分。
