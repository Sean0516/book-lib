# 深度精选

原有 520 道题负责知识广度，这里负责工程深度。每篇文章先给出可直接口述的
`60–90 秒速答`，再展开机制、证据、代码、权衡、指标和三级追问。

## 当前模块

### [JVM 与并发](./jvm-concurrency/)

从 GC、内存泄漏、线程池、锁竞争、JMM 到 CPU 飙高排查，最后通过一个 Full GC 教学
案例把六个主题串成完整证据链。

## 按岗位学习

| 路线 | 当前建议 | 重点能力 |
| --- | --- | --- |
| 高级 Java 后端 | JVM 与并发全部 6 题 → Full GC 案例 | 原理、代码、排障 |
| 架构师/技术负责人 | GC 选型 → 线程池容量 → 锁竞争 → Full GC 案例 | 约束、权衡、指标 |
| AI 架构师 | 线程池容量 → CPU 排查 → Full GC 案例 | AI 网关和工具执行的 JVM 底座 |

后续会按质量验收结果增加数据库、缓存与消息、容量规划、分布式一致性、稳定性、RAG 和
Agent；不会为了数量一次性复制同一种模板。

## 你会在文章里看到什么

- **图**：机制图、时序图和排查决策树。
- **代码**：能暴露边界和风险的最小 Java、命令或配置。
- **案例**：带输入数据、时间线、证据与回滚方案的教学场景。
- **指标**：说明定义、来源、示例基线和决策动作。
- **评分**：正确性、深度、权衡、表达和可运维性，共 25 分。
