# 数据与消息可靠性

这组专题不重复背概念，而是把数据库、缓存和消息三条链路连接起来：数据库是事实源，缓存
提供读取加速，消息承载异步传播。任何一层只讲正常流程，都不足以回答高级面试。

## 三个核心专题

1. [数据库问题如何从证据走到低风险变更？](./01-database-evidence-chain)
2. [缓存一致性、热点与故障回源如何一起设计？](./02-cache-consistency-and-hotspots)
3. [消息不丢、不重、不乱和不积压如何系统回答？](./03-messaging-reliability)

## 推荐顺序

```text
数据库事实源 → 缓存读取副本 → 消息变更传播 → 对账与修复
```

每篇先读 90 秒答案，再根据自己的薄弱点进入证据、方案取舍和面试官追问。最后使用配套案例
复述一次完整事故，不建议只阅读参考答案。

## 完成标准

- [ ] 能区分观察、推断和最终结论。
- [ ] 能给出至少一个容量计算或时间预算。
- [ ] 能说明方案没有解决的剩余风险。
- [ ] 能定义越线指标、止血动作和回滚条件。
- [ ] 能从数据库、缓存、消息三层检查一致性闭环。
