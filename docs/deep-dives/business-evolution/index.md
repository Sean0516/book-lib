# 业务建模与系统演进

这一模块不要求把所有业务都“DDD 化”，而是训练如何找到变化边界、保护业务不变量，并在不中断
业务的前提下逐步替换遗留系统。

## 学习地图

| 专题 | 核心问题 | 面试产出 |
| --- | --- | --- |
| [领域发现与限界上下文](./01-domain-discovery) | 边界从哪里来 | 事件、语言、能力、上下文映射 |
| [聚合、不变量与领域事件](./02-aggregate-invariants) | 什么必须同事务，什么可以最终一致 | 聚合边界、命令、事件、补偿 |
| [单体渐进迁移](./03-strangler-migration) | 第一刀怎么切且能回退 | 绞杀路径、影子流量、迁移门禁 |
| [遗留系统与模型腐化治理](./04-legacy-modernization) | 如何控制反腐层、数据和组织债务 | 事实源、ACL、退场与收益 |

## 演进闭环

```mermaid
flowchart LR
  A["业务事件与变化原因"] --> B["限界上下文"]
  B --> C["聚合与不变量"]
  C --> D["服务/模块所有权"]
  D --> E["渐进迁移与兼容"]
  E --> F["交付、稳定性与业务指标"]
  F --> A
```

## 完成标准

- 能用业务语言、规则和变化原因解释边界，而不是按数据库表拆服务。
- 能明确强一致不变量的最小范围和跨域收敛方式。
- 能给单体拆分设计流量、数据、回滚和旧路径退场。
- 能用交付速度、故障隔离、跨团队等待和数据差异验收拆分。
