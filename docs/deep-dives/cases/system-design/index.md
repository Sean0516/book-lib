# 系统设计完整案例

训练目标是把入口流量、服务边界、数据一致性、故障隔离和可观测性放进同一决策框架。


## 可反向回答的题库问题

系统设计案例用来训练从“单点知识”到“完整架构方案”的表达：

- 秒杀系统如何做容量估算、限流削峰、库存扣减和异步化？
- 高并发订单系统中，哪些链路必须同步，哪些链路可以异步？
- 级联故障为什么常由超时、重试和连接池耗尽共同放大？
- 熔断、限流、降级、隔离各自解决什么问题，顺序怎么排？
- 如何把可观测性、压测、预案和复盘纳入架构设计，而不是事后补丁？
