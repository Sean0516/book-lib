# Redis 完整案例

训练目标是区分 Redis 自身瓶颈、访问分布问题和缓存失效对数据库的放大效应。


## 可反向回答的题库问题

Redis 案例的重点不是“缓存很快”，而是如何在高并发下控制热点、一致性和降级边界：

- 热 Key 如何发现？是扩容、拆 Key、本地缓存还是限流？
- 缓存击穿、穿透、雪崩分别是什么，治理手段为什么不同？
- 缓存和数据库不一致时，如何判断是删除策略、并发写还是消息补偿问题？
- 多级缓存会带来哪些一致性、失效和容量风险？
- Redis 故障时，系统如何优雅降级，而不是把数据库打穿？
