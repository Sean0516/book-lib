# 数据库完整案例

训练目标是先用工作负载和数据库证据确认瓶颈，再选择最低风险修复，最后说明扩展边界。


## 可反向回答的题库问题

这些数据库案例适合把“索引、事务、锁、分库分表”从概念拉到现场：

- 慢 SQL 是先改 SQL、加索引，还是改业务读模型？判断依据是什么？
- Explain 里 type、rows、filtered、Extra 分别怎么影响优化判断？
- 死锁和普通锁等待有什么区别？如何快速止血？
- 事务隔离级别、加锁顺序和索引选择为什么会影响并发写？
- 高并发订单存储如何兼顾写入吞吐、查询效率、幂等和扩展性？
