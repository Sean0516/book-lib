# 面试题库深度升级方案

日期：2026-07-14  
目标岗位：高级 Java 后端、架构师、AI 架构师  
升级原则：保留 520 题的覆盖广度，只对高价值问题增加深度，避免把题库改成长篇百科。

## 一、内容分级

### L1：快速定位题

适合定义、对比和基础原理题，目标是 30–60 秒讲清。

必须包含：

1. 一句话结论。
2. 3–5 个关键点。
3. 2–3 个高频追问。
4. 版本边界或易错点。

不强制加入架构扩展、指标清单和案例，避免为简单问题制造模板内容。

### L2：高频深度题

适合 JVM、数据库、分布式、稳定性、RAG、Agent 等高频主题，目标是能承受三轮追问。

必须包含：

1. 60–90 秒标准回答。
2. 原理或关键执行链路。
3. 一个带输入数据的微型场景。
4. 至少两种方案及取舍。
5. 失败模式、降级或回退动作。
6. 指标定义、示例基线和触发动作。
7. 三级追问与 25 分评分点。

### L3：完整场景题

适合系统设计、线上事故和复杂 AI 架构题，目标是形成可复述的项目级答案。

必须包含：

1. 业务目标和约束：QPS、数据规模、TP99、可用性、一致性、成本或团队条件。
2. 高层架构和关键数据/控制流。
3. 容量估算或成本估算。
4. 依赖超时、重试风暴、热点、重复请求、部分成功等故障推演。
5. 备选方案和未选择原因。
6. 灰度、回滚、对账、演练和验收阈值。
7. 五轮追问与 25 分评分表。

## 二、统一验收标准

每道 L2/L3 题按以下五项各 0–5 分验收：

| 维度 | 5 分标准 |
| --- | --- |
| 正确性 | 概念准确，并标注版本或实现边界 |
| 深度 | 能解释机制、关键链路和失败原因 |
| 取舍 | 至少比较两种方案，说明适用条件和剩余风险 |
| 表达 | 结论先行，90 秒回答可直接口述 |
| 可运维性 | 指标有口径、示例阈值、验证和回退动作 |

交付门槛：

- L2 不低于 20/25。
- L3 不低于 22/25。
- 示例数字必须注明是教学假设，不能包装成通用标准。
- 不允许只列中间件名称而不说明解决的问题。
- 最终一致性方案必须包含补偿或对账。
- 每个新增深度专题必须能从题库和深度精选两个入口访问。

## 三、分批路线

### 第 1 批：现代 Java、JVM 与并发

范围：

- 修正 JVM 内存区域和 HotSpot 实现边界。
- 修正偏向锁的 JDK 版本表述。
- 升级线程池容量题。
- 新增 Java 21 虚拟线程深度专题。
- 更新抽认卡、导航和深度精选入口。

验收：至少 3 道 L2 修订、1 篇 L3 专题、构建通过。

状态：已完成。

实际交付：

- 升级 JVM 运行时数据区和 Metaspace 两道题，补齐规范、HotSpot 和进程内存边界。
- 升级 `synchronized`，补充 JDK 15 偏向锁版本边界。
- 升级线程池容量题，加入 Little's Law、微型数据、方案取舍、故障压测和回退。
- 新增 Java 虚拟线程 L3 专题，覆盖 JDK 21/24、资源门控、迁移风险和 25 分自评。
- 更新高频抽认卡、深度精选索引和侧栏导航。

### 第 2 批：数据库、缓存与消息

优先题：

- 索引设计、MVCC、慢 SQL、死锁。
- 缓存一致性、热点 Key、击穿与雪崩。
- 消息不丢、重复消费、乱序、积压。

交付：20 道 L2；补强 5 个现有 L3 案例；建立 SQL/Redis/MQ 证据模板。

### 第 3 批：分布式与稳定性

优先题：

- 幂等、分布式事务、最终一致性、分布式锁。
- 超时、重试、熔断、隔离、降级。
- SLO、错误预算、RTO/RPO、多活与容灾。

交付：25 道 L2；新增或补强 8 道 L3；加入故障传播和恢复验证。

### 第 4 批：系统设计与技术治理

优先题：

- 容量规划、秒杀、订单、搜索、通知和多租户。
- 服务边界、数据分片、迁移、灰度和回滚。
- 架构决策、成本、组织边界和演进触发点。

交付：25 道 L2；10 道可完整模拟的 L3 系统设计题。

### 第 5 批：AI 架构

优先题：

- 模型选型和路由。
- RAG 切块、混合检索、重排、权限和评测。
- Agent/Workflow 边界、工具协议、权限、重试和审计。
- AI 服务的延迟、成本、安全、实验与 ROI。

交付：30 道 L2；10 道 L3；所有时效性内容附来源或更新时间。

### 第 6 批：学习体验与总体验收

范围：

- 高频 100 题与升级后的 L2/L3 对齐。
- 为核心题加入“先答后看”、自评分和错题复习入口。
- 清理模板化表达和重复题。
- 检查所有跨模块链接、导航、移动端阅读和构建结果。

## 四、每批工作流

1. 选题：按高频、区分度、时效风险和现有案例覆盖排序。
2. 基线审计：记录原答案得分与具体缺口。
3. 小样升级：先完成 3–5 题，统一措辞和结构。
4. 批量升级：只扩写符合 L2/L3 条件的题。
5. 反向追问：用面试官视角连续追问三到五轮。
6. 构建检查：验证 Markdown、Mermaid、链接和 VitePress 构建。
7. 进度记录：更新本文件的批次状态和变更清单。

## 五、总体目标

- 保留约 350 道 L1 快速题。
- 建成约 120 道 L2 高频深度题。
- 建成约 50 道 L3 场景题，其中复用并升级现有案例。
- 核心内容覆盖高级 Java、架构和 AI 三条面试路线。
- 用户可以在 30 秒内定位题目，在 90 秒内练习回答，并继续进入完整案例。
